# Contract: Product Roadmap Packet

The roadmap should be staged, not a fixed calendar promise.

## Recommended stages

- Stage 0 — Discovery / concept validation
- Stage 1 — MVP
- Stage 2 — Validation / soft launch
- Stage 3 — Public launch
- Stage 4 — Retention / engagement
- Stage 5 — Monetization expansion
- Stage 6 — Scale / live operations

## Each stage should include

- goal;
- hypothesis;
- must-have scope;
- explicit non-goals;
- metrics;
- validation approach;
- exit criteria;
- risks;
- downstream technical implications.

## Token-aware storage

Store roadmap as:

```text
docs/product/roadmap/index.md
docs/product/roadmap/stages/01-mvp.md
...
```

Do not require Opus to read future stages during normal task execution.
