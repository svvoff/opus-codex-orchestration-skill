# Index-First Reading

Before reading detailed files, read index files that route to the smallest relevant context.

## Examples

- Read `docs/product/backlog/index.md` before opening task cards.
- Read `docs/product/roadmap/index.md` before opening roadmap stages.
- Read `docs/product/decisions/index.md` before opening decision cards.
- Read `docs/ai/validation/index.md` before opening validation profiles.

## Why

Index files allow Opus to select context rather than ingest everything.

## Index quality rules

An index should be:

- short;
- current;
- routing-oriented;
- not a duplicate of all file contents;
- explicit about what not to read by default.
