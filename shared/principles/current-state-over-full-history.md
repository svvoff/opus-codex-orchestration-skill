# Current State Over Full History

Use `docs/product/current-state.md` as the compact source of current project truth.

It should contain:

- current stage;
- current focus;
- active constraints;
- current risks;
- active non-goals;
- next task source;
- links to indexes.

Update it when the active milestone, focus, or constraints change.

Do not read the full project brief or full roadmap for every execution cycle unless current state is insufficient.
