# Draft First, Write After Approval

Skills must not write generated configuration or product documents immediately.

Required behavior:

1. Build a draft.
2. Show assumptions, file list, and generated content summary.
3. Ask the user for review.
4. Write files only after explicit approval.

This applies to:

- product discovery docs;
- roadmap/backlog files;
- reusable profile drafts;
- `CLAUDE.md`;
- `AGENTS.md`;
- validation and execution policy files.
