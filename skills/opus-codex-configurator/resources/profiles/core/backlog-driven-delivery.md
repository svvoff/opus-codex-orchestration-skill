# Profile: Backlog-Driven Delivery

## Category

core/delivery

## When to use

When the target project has product backlog/task cards.

## Opus rules

- Select tasks from backlog, not memory.
- Check dependencies and risk.
- Convert selected task to Task Packet.
- Update status after review.

## Codex rules

- Execute only the provided task packet.
- Do not choose or reprioritize tasks.

## Validation requirements

Every task must define validation requirements before it can be `ready`.
